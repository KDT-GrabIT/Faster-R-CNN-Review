1. R-CNN (1/5): 등장 배경

"이미지를 다 보지 말고, 물체가 있을 곳만 보자"

2014년 이전에는 이미지 전체를 다양한 크기의 박스로 훑는 Sliding Window 방식을 사용했습니다. 이는 연산량이 너무 많고 비효율적이었습니다.

💡 핵심 아이디어: Region Proposal (영역 제안)
"물체가 있을 법한 위치(RoI)를 먼저 2,000개 정도 찾고, 그 부분만 집중적으로 CNN에 넣어보자!"

R-CNN 파이프라인

  1. 입력 이미지 준비
  2. Region Proposal (약 2,000개 추출)
  3. 각 영역별 CNN 특징 추출 (개별 수행)
  4. SVM 분류 및 BBox Regression

1. R-CNN (2/5): Selective Search란?

Selective Search: 작은 영역들을 합쳐나가는 과정

후보 영역을 찾는 알고리즘

R-CNN은 딥러닝이 아닌 Selective Search라는 고전적 컴퓨터 비전 알고리즘을 사용합니다.

  • 원리: 이미지를 색상, 무늬, 크기가 비슷한 픽셀끼리 잘게 쪼갠 후(Segmentation), 유사한 인접 영역들을 반복적으로 합쳐나갑니다(Grouping).
  • 결과: 최종적으로 물체라고 추정되는 약 2,000개의 박스 좌표를 반환합니다.
🍕 비유: 흩어진 퍼즐 조각들을 색깔별로 모아서 "이 뭉치는 자동차 같네?", "저 뭉치는 사람 같네?" 라고 짐작하는 것과 같습니다.

1. R-CNN (3/5): Warping (강제 변환)

이미지를 억지로 찌그러뜨린다?

Selective Search가 찾아낸 2,000개의 박스는 크기가 제각각입니다 (길쭉한 것, 넓은 것 등). 하지만 당시 CNN(AlexNet)은 고정된 입력 크기(227x227)만 받을 수 있었습니다.

  • Warping: 잘라낸 이미지의 비율을 무시하고 강제로 정사각형(227x227)으로 변환합니다.
  • 문제점: 이미지가 찌그러지면서 정보 손실과 왜곡이 발생해, 모델이 물체를 제대로 인식하지 못하는 원인이 됩니다.

Warping에 의한 이미지 왜곡 (이미지: warping.jpg)

1. R-CNN (4/5): 2,000번의 CNN 연산

무식한 계산량 (Brute Force)

R-CNN의 가장 큰 비효율성은 잘라낸 2,000개의 이미지를 각각 하나씩 CNN 모델에 통과시킨다는 점입니다.

⛔ 극악의 속도 (47초/장)
이미지 한 장을 처리하는 데 47초가 걸립니다. 학습 과정도 복잡하여(CNN, SVM, Regressor 따로 학습), 84시간 이상의 긴 학습 시간이 필요했습니다.
🍕 비유: 배달지 2,000곳을 가야 하는데, 한 곳 배달하고 가게로 돌아왔다가 다시 출발하기를 2,000번 반복하는 것과 같습니다. (시간과 자원 낭비)

X 2,000 Iterations

1. R-CNN (5/5): 한계점 정리

왜 R-CNN으로는 안 되는가?

  • 속도: 47초/장은 실시간 서비스에 적용 불가능합니다.
  • 구조: Multi-stage Pipeline (CNN, SVM, Regressor가 따로 놈)이라 학습과 관리가 어렵습니다.
  • 용량: 추출된 특징(Feature)을 디스크에 저장해야 해서 수백 GB의 공간이 필요합니다.

→ "이 모든 문제를 해결하기 위해 Fast R-CNN이 등장합니다."

2. Fast R-CNN (1/3): 속도 혁명

"CNN을 2,000번 돌리지 말고, 딱 1번만 돌리자!"

R-CNN의 무식한 연산 중복 문제를 해결하기 위한 획기적인 아이디어입니다.

  • 기존 (R-CNN): 이미지 자르기(Crop) → 각각 CNN 통과 (2,000번)
  • 혁신 (Fast R-CNN): 이미지 전체 CNN 통과 (1번) → Feature Map 생성 → 여기서 필요한 부분만 좌표로 가져오기

이미지 전체를 한 번에 처리하는 구조 (이미지: fast_rcnn_structure.jpg)

2. Fast R-CNN (2/3): RoI Pooling

RoI Pooling: 가변 입력을 고정 출력으로 변환 (이미지: roi_pooling.jpg)

찌그러뜨리지 않고 크기 맞추기

Feature Map에서 잘라낸 영역들은 크기가 제각각입니다. 이를 해결하기 위해 RoI(Region of Interest) Pooling 기술이 도입되었습니다.

🛠️ RoI Pooling 역할
어떤 크기의 영역이 들어와도 Max Pooling을 이용하여 고정된 크기(예: 7x7)의 Feature Map으로 변환해줍니다. 이로써 Warping으로 인한 정보 손실 문제를 해결했습니다.

2. Fast R-CNN (3/3): 여전한 병목 현상

"엔진은 페라리인데, 네비게이션이 종이 지도"

CNN 연산 속도는 0.3초로 획기적으로 줄었지만, 전체 시스템 속도는 여전히 느렸습니다.

🛑 Selective Search (CPU)의 한계
후보 영역을 제안하는 알고리즘(Selective Search)이 여전히 CPU에서 돌아가며 2초나 잡아먹습니다. 딥러닝 모델이 아무리 빨라져도, 이 전처리 과정 때문에 실시간 구동은 불가능합니다.

3. Faster R-CNN (1/3): RPN의 등장

"후보 영역 찾는 것도 GPU가 하게 하자!"

병목의 원인인 Selective Search를 제거하고, 그 역할을 수행하는 별도의 신경망 RPN (Region Proposal Network)을 도입했습니다.

🚀 RPN (Region Proposal Network)
Feature Map을 입력받아, 물체가 있을 법한 위치(Bounding Box)와 그 확률(Objectness Score)을 순식간에 계산합니다. CNN 연산을 공유하기 때문에 속도가 거의 0초(Cost-free)에 가깝습니다.

Selective Search가 RPN으로 대체된 구조 (이미지: faster_rcnn_structure.jpg)

3. Faster R-CNN (2/3): Anchor Box

다양한 비율의 Anchor Box 예시 (이미지: anchor_box.jpg)

맨땅에 헤딩하지 않는다: Anchor Box

RPN은 이미지를 훑을 때, 각 지점마다 미리 정의된 9개의 틀(Anchor Box)을 대봅니다.

  • 3가지 크기: 128, 256, 512
  • 3가지 비율: 1:1, 1:2, 2:1 (정사각형, 길쭉한 것, 납작한 것)

이 틀을 기준으로 "이 안에 물체가 있는가?"와 "틀을 얼마나 수정해야 딱 맞는가?"를 학습하여, 다양한 형태의 물체를 효율적으로 찾아냅니다.

3. Faster R-CNN (3/3): 진정한 End-to-End

모든 것이 GPU 안에서 (Fully Convolutional)

Faster R-CNN은 특징 추출(ResNet), 영역 제안(RPN), 분류(Classifier)가 모두 하나의 통합된 네트워크로 연결되어 있습니다.

🚀 비유: 이제 굼뜬 조수는 없습니다. 태블릿(GPU)에 설치된 AI 내비게이션(RPN)이 켜자마자 맛집 위치를 순식간에 찍어줍니다. 배달부는 멈추지 않고 질주합니다.
  • 속도: 0.2초 (5 FPS). 드디어 실시간 처리에 근접했습니다.
  • 정확도: RPN도 함께 학습되므로 후보 영역의 품질이 매우 높습니다.

Total Speed: 0.2s

4. 최종 Faster R-CNN 구조도

Step 1. RPN
Feature Map에서 Anchor Box를 이용해 물체 후보(Proposal)를 찾습니다.
Step 2. Fast R-CNN Detector
찾은 후보를 RoI Pooling으로 크기를 맞추고, 최종 클래스와 좌표를 확정합니다.

3. 실험 설계: 왜 YOLO와 비교하는가?

비교 목적 (Purpose)

🧐 정확도 vs 속도 (Accuracy vs Speed)
Faster R-CNN은 2-Stage 구조로 매우 높은 정확도를 자랑하지만, 실시간 서비스에 적용하기엔 너무 무겁습니다.
반면, YOLO는 1-Stage 구조로 압도적인 속도를 자랑합니다.

YOLO: 1-Stage Detector란?

"You Only Look Once" (한 번만 보면 된다)

  • Faster R-CNN (2-Stage): "어디 있지?"(RPN) 찾고 → "이게 뭐지?"(Classifier) 확인 (두 번 일함).
  • YOLO (1-Stage): 이미지를 한 번 쓱 보면서 위치와 종류를 동시에 맞춥니다. (한 번만 일함 → 빠름)

검증 포인트

  • Faster R-CNN: "느려도 정확한가?" (Baseline)
  • YOLOv5s: "빠른데도 정확한가?" (Challenger)

4. 최종 결론 (Conclusion)

모델 선정 가이드

  • 상황 A: 정밀도 최우선 (Faster R-CNN)
    의료 영상, 정밀 불량 검출 등 0.1%의 오차도 허용하지 않는 경우
    → Faster R-CNN 추천
  • 상황 B: 실시간성 최우선 (YOLOv5s)
    자율주행, 모바일 AR, 시각장애인 보조 등 즉각적인 반응이 생명인 경우
    → YOLOv5s 추천
🏆 최종 결론
최신 경량화 모델(YOLOv5s)이 기술의 발전 덕분에 과거의 고성능 모델(Faster R-CNN)의 정확도를 거의 따라잡았으며, 속도 면에서는 비교 불가능한 우위를 점하고 있음을 확인했습니다.
